== Introduction and Goals

Resource manager, shortly RM, provides mechanisms for hierarchical resources locking.
It was created to support external agents that need to get exclusive access to shared network resources (for configuration purposes, for example) and in order to avoid concurrency and misconfiguration only one agent should be able to gain access.

The main goal of RM is to provide configurable API what allows mutual exclusiveness for resources in hierarchical way.

The current solution MOOSE is missing sophisticated locking functionality and solves resource contention problem by simply retrying mechanism.
